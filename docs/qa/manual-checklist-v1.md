**Manual QA Checklist – PicDay (Write/Diary Refactor v1)**

- [ ] Calendar에서 날짜 선택 → Write 진입 시 모드가 올바르게 결정된다 (기록 없음: ADD / 기록 있음: VIEW)
- [ ] Write VIEW 화면에서 기존 기록 목록이 정상 표시된다
- [ ] Write VIEW에서 “기록 추가” → ADD 화면으로 전환된다
- [ ] Diary 화면에서 “이 날의 기록 더 추가하기” → ADD 화면으로 전환된다
- [ ] Diary 화면에서 기록 목록이 날짜와 일치한다 (날짜 변경 후 반영 확인)

- [ ] ADD 화면에서 제목/내용 입력 후 저장 → 목록에 새 기록이 추가된다
- [ ] ADD 화면에서 사진 1장 추가 → 썸네일 표시됨 → 저장 후 유지된다
- [ ] ADD 화면에서 사진 여러 장 추가 → 썸네일 표시됨 → 저장 후 유지된다
- [ ] ADD 화면에서 사진 추가 후 개별 삭제 → 삭제한 사진은 저장 후 반영되지 않는다
- [ ] ADD 화면에서 사진 전부 삭제 후 저장 → 기록은 저장되며 사진은 0장이다

- [ ] VIEW에서 기록 선택 후 EDIT 진입 → 기존 제목/내용이 로드된다
- [ ] EDIT 진입 시 기존 사진이 모두 로드된다
- [ ] EDIT에서 사진 일부 삭제 → 저장 후 삭제된 사진만 제거된다
- [ ] EDIT에서 사진 전부 삭제 → 저장 후 사진 0장 상태로 유지된다
- [ ] EDIT에서 사진 유지 + 신규 추가 혼합 → 저장 후 유지/추가 모두 반영된다
- [ ] EDIT에서 텍스트만 수정 → 저장 후 텍스트만 변경되고 사진은 유지된다

- [ ] 어떤 모드에서든 뒤로가기 → 이전 화면으로 정상 복귀된다
- [ ] 저장 후 자동 복귀 동작이 정상이다
- [ ] 저장 후 복귀된 화면에서 목록/상세가 최신 상태로 보인다
