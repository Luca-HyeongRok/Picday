# PicDay PRD v1

## 1. 제품 목표 (Product Goal)
- 문제: 모바일에서 날짜별 일기를 빠르게 기록/조회하기 어렵고, 사진이 많은 기록은 관리가 번거롭다.
- 핵심 가치: 캘린더 기반으로 날짜를 즉시 찾고, 텍스트+사진 기록을 간단하게 추가/수정/삭제할 수 있게 한다.

## 2. 타겟 사용자 (Target User)
- 일정/일상을 자주 기록하는 개인 사용자
- 사진이 포함된 일기를 간편하게 관리하고 싶은 사용자

## 3. 현재 완료 범위 (Current Scope)
- Room 영속성 + Hilt 연동
- debug 빌드: InMemory Room + seed data
- release 빌드: 실제 Room DB 사용
- WriteScreen 다중 사진 선택 지원
  - 썸네일 미리보기
  - 개별 삭제 버튼
  - 저장 시 `photoUris: List<String>` ViewModel로 전달

## 4. 다음 개발 범위 (Next Scope)
1) 다이어리 편집 모드
- 기존 텍스트 및 사진 로딩
- 사진 유지/삭제 처리

2) 사진 영속성 전략 정리
- `photoUri` 저장 방식 명확화
- Room 스키마 반영 기준 정의

3) 삭제 정책 정의
- 다이어리 삭제 시 사진 처리 기준 확정
- DB/파일 레벨 삭제 동작 정의

4) 마이그레이션 전략
- 사진 컬럼/테이블 변경 대비
- 데이터 보존 기준 및 Migration 방향 정의

## 5. 제외 범위 (Out of Scope)
- 카메라 촬영 기능 추가
- 썸네일 캐싱 및 미디어 성능 최적화
- 고급 검색/태그/필터
- 클라우드 동기화

## 6. 수용 기준 (Acceptance Criteria)
- 편집 화면 진입 시 기존 텍스트/사진이 정상 로딩된다.
- 편집 화면에서 사진을 유지/삭제할 수 있고, 저장 시 변경 사항이 정확히 반영된다.
- `photoUri` 저장 정책이 문서화되어 있으며, Room 스키마에 반영된다.
- 다이어리 삭제 시 사진 처리 정책(삭제/유지)이 일관되게 적용된다.
- Room 마이그레이션 전략이 정의되어 있으며, 버전 업 시 데이터 손실 없이 동작한다.
